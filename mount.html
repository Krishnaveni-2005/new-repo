Linux Mount Command

7. mount

Description

The mount command in Linux is used to attach file systems to the directory tree, making them accessible. It is often used for mounting storage devices such as hard drives, USB drives, or network shares to specific directories.

Purpose and Usage

The mount command enables you to access data from devices such as USB drives, network storage, and other partitions. It connects the storage device to the file system so that it can be accessed like any other directory or file on your system.

Syntax:

mount [options] [device] [mount_point]

Examples

To mount a USB drive (e.g., /dev/sdb1) to a directory (e.g., /mnt/usb):

sudo mount /dev/sdb1 /mnt/usb

To mount a network file system (e.g., //server/share):

sudo mount -t cifs //server/share /mnt/network

Additional Explanation

The mount command is critical for accessing different types of storage on a Linux system. Some common options include:

  • -t: Specify the file system type (e.g., ext4, ntfs, vfat)
  • -o: Pass specific options, such as rw (read-write) or ro (read-only)
  • loop: Use this for mounting disk image files (e.g., ISO files)

Unmounting is equally important. Once you are done with the device, you should unmount it to ensure that all data is written to the device properly. You can unmount using the umount command:

sudo umount /mnt/usb

The mount command is essential for working with external devices, network shares, and various types of file systems in Linux.
